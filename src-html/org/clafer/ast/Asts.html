024    // Not going to define group cardinality key words since they are confusing
025    // and they don't work like they do in the Clafer compiler. For example:
026    //   opt A
027    //       B
028    //       C
029    // is desugarred in the Clafer compiler to
030    //   opt A
031    //       B ?
032    //       C ?
033    // So the keywords implicitly affect the cardinalities of the children. If
034    // these keywords were defined in the API as below, they do not desugar the
035    // children and the behaviour is unexpected. Hence leave them undefined.
036    // public static final Card Opt = new Card();
037    // public static final Card Mux = new Card(0, 1);
038    // public static final Card Xor = new Card(1, 1);
039    // public static final Card Or = new Card(1);
